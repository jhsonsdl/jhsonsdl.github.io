我的第一个hexo博客
发布于 2019-07-13   更新于 2019-07-14 热度 0 喜欢 0

其实搭建一个博客有很多种方式，但是我选择了Hexo+GitHub的方式，初次使用，结果就是四处碰壁，不过搭建的过程其实也并不难，只要照着文档或者网上的一些大神博客的教程也是可以很轻松的搭建出来的，(https://blog.csdn.net/sinat_37781304/article/details/82729029 “参照博客”)

这边主要分两个部分：

  1. hexo的初级搭建还有部署到github page上，以及个人域名的绑定。
  2. hexo的基本配置，更换主题。

Hexo搭建步骤

  1. 安装Git
  2. 安装Node.js
  3. 安装Hexo
  4. GitHub创建个人仓库
  5. 生成SSH添加到GitHub
  6. 将hexo部署到GitHub
  7. 设置个人域名
  8. 发布文章

4.GitHub创建仓库

首先要去注册一个GitHub账号因为github都是因为如果看不懂的话可以使用goole浏览器打开，因为goole浏览器有个翻译因为的功能，其他浏览器可能也有不过我不知道。
登录自己的github后去创建一个仓库（New repository）创建一个和你用户名相同的仓库，后面加.github.io，只有这样，将来要部署到GitHub page的时候，才会被识别，也就是xxxx.github.io，其中xxx就是你注册GitHub的用户名。点击create repository。

这个时候它会告诉你已经生成了.ssh的文件夹。在你的电脑中找到这个文件夹。这个文件一般在(C:\Users\Administrator.ssh)这个用户的目录下

ssh，简单来讲，就是一个秘钥，其中，id_rsa是你这台电脑的私人秘钥，不能给别人看的，id_rsa.pub是公共秘钥，可以随便给别人看。把这个公钥放在GitHub上，这样当你链接GitHub自己的账户时，它就会根据公钥匹配你的私钥，当能够相互匹配时，才能够顺利的通过git上传你的文件到GitHub上。

而后在GitHub的setting中，找到SSH keys的设置选项，点击New SSH key
把你的id_rsa.pub里面的信息复制进去。

然后在gitbash中查看是否成功如果对powershell不熟前面那些命令就都用gitbash执行

7. 设置个人域名

现在你的个人网站的地址是 yourname.github.io，如果需要域名可以花几块钱或者10多块钱去买一个。

你需要先去进行实名认证,然后在域名控制台中，看到你购买的域名，点解析进去，添加解析。
